  • ⊕ Load average, iowait e interrupciones del kernel.

[04] · terminal segura

SSH auditada,
no SSH libre.

La terminal integrada no es un shell abierto. Es una interfaz de comandos con whitelist definida por equipo y rol. Lo que no esté autorizado se bloquea y se registra con timestamp, usuario y servidor.

  • ⊕ Whitelist por rol: ops, lectura, on-call.
  • ⊕ Soporte para systemctl, journalctl, ss, df.
  • ⊕ Sesiones grabadas, exportables a JSON.
  • ⊕ Rotación de claves SSH ed25519 desde el panel.

[09] · preguntas frecuentes

Lo que siempre nos preguntan.

¿En qué hardware puedo desplegar Zyntra? +

Cualquier servidor o equipo encendido y conectado a tu red: Linux, Windows, macOS, una VM, un VPS privado o una Raspberry Pi. El requisito es Python 3.8+, 512 MB de RAM y conectividad SSH hacia los nodos Linux que querés monitorear. El instalador es un solo script curl | sh (o el ejecutable nativo en Windows) que configura Python, SQLite, claves ed25519 y el servicio del sistema.

¿Cómo funciona la seguridad del acceso SSH a los servidores? +

Zyntra usa claves ed25519 generadas en el primer arranque. Nunca almacena contraseñas. Recomendamos crear un usuario de sistema dedicado en cada servidor Linux con permisos mínimos (AllowUsers zyntra en sshd_config). La terminal integrada solo ejecuta comandos de la whitelist configurada por rol. Cada sesión queda registrada con timestamp, usuario y output completo.

¿Con qué distribuciones Linux es compatible? +

Los servidores monitoreados deben correr Linux: Debian 10+, Ubuntu 20.04+, Rocky Linux 8+, AlmaLinux 8+, RHEL 7+, CentOS 7+, Arch o Alpine. El colector usa /proc, /sys y comandos POSIX (ss, df, journalctl, ps). El servidor donde corre Zyntra, en cambio, puede ser cualquier OS con Python 3.8+: Linux, Windows o macOS.

¿Qué impacto tiene en los servidores Linux monitoreados? +

Prácticamente cero. Cada ciclo de recolección ejecuta lecturas de /proc/stat, /proc/meminfo y df vía SSH — menos de 0.05% de CPU en servidores modernos. El intervalo de polling es configurable: 2 segundos por defecto, ajustable hasta 60 segundos. No se instala ningún proceso permanente ni daemon en los servidores monitoreados.

¿Puedo conectar Zyntra a Slack, Teams o mi sistema de alertas? +

Sí. Las alertas se enrutan a un webhook genérico (JSON configurable), SMTP o cualquier endpoint HTTP. Hay plantillas listas para Slack, Teams, Mattermost, Rocket.Chat y PagerDuty en la guía de despliegue. En Enterprise, la API REST expone todos los endpoints de métricas históricas para integración con Grafana u otros sistemas internos.
